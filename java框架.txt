# EhCache缓存
* [如果是单个应用或者对缓存访问要求很高的应用，用ehcache.如果是大型系统，存在缓存共享、分布式部署、缓存内容很大的，建议用redis,ehcache也有缓存共享方案，不过是通过RMI或者Jgroup多播方式进行广播缓存通知更新，缓存共享复杂，维护不方便；简单的共享可以，但是涉及到缓存恢复，大数据缓存，则不合适。工作方式不一样，可以理解为ehcache是单机版缓存，redis是服务器版的缓存.]()
